El formulario de registro debe tener el method="POST"  y los input deben tener los name="" y los id="" (Indentificadores). Tambien el formulario debe esta conectado a registerController.php en el action de formulario  form action="../../controllers/auth/registerController.php" method="POST" id="formRegister". 

El registerController.php se encarga de recibir los datos del formulario de registro y enviarlos al modelo Usuario.php para que se guarden en la base de datos.

En registerController.php esta conectado al modelo Usuario.php con  require_once __DIR__ . '/../../models/Usuario.php';. Inicial debe tener session_start(); para que se pueda crear la sesion. luego relaciona los name de los input del formulario de registro con los name de las variables en registerController.php. Por ejemplo, si en el formulario de registro el input tiene el name="usuario", en registerController.php se debe relacionar con la variable $usuario. Igualmente se debe relacionar la variable del metodo POST $_POST relacionandolo con un nueva variable en registerController.php. Por ejemplo, si en el formulario de registro el input tiene el name="usuario", en registerController.php se debe relacionar con la variable $usuario = $_POST['usuario'] ?? ''; .

El modelo Usuario.php es el que se encarga de guardar los datos en la base de datos. Tiene que terner una funcion registrar($usuario, $email, $password, $rol) que reciba los datos del formulario de registro y los guarde en la base de datos. Debe tener una funcion emailExiste($email) que verifique si el email ya existe en la base de datos. Recuerde que este modelo hace referencia a la taabla de la base de datos llamada 'usuarios' y tiene los campos 'id', 'usuario', 'email', 'password' y 'rol'.
